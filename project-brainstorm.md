I would like to port the [keyd](https://github.com/rvaiya/keyd) keyboard layout I have defined in @keyd_enthium_layout.conf into [kanata](https://github.com/jtroo/kanata).

I first wrote this layout because I'd been using sunaku's glorious engrammer layout on the moergo glove80 keyboard. This layout has several features that I found essential and wanted to replicate on my regular laptop keyboard:
- The symbol layer, which is present in the aforementioned keyd layout
- The number layout, also present in the keyd layout
- home-row-mods (miryoku layers). these are present in the keyd layout... but are not as comfortable to use due to the lack of typing streaks.
- Bilateral combination typing streaks. These are not present in my keyd layout as they would take a very sophisticated implementation that keyd does not support. I've been led to believe, however, that kanata would support such functionality.

I made my best effort with the tools keyd affords, but I'd really like to figure out a real implementation. I do not want to enforce opposite hand combos.

You can read sunaku's post about streaks and bilateral combos here: https://sunaku.github.io/home-row-mods.html (keep in mind his glove80 layout has gone through ~2 years of updates since then.)

How might we go about approaching this?
